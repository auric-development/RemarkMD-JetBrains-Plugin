Why this file is checked in
--------------------------
RemarkMD renders documents offline, in a sandboxed WKWebView with no page network
access. The `dist/mermaid.min.js` build is a standalone IIFE with every diagram
type inlined and no dynamic import(), so it runs with nothing to fetch. Mermaid's
smaller ESM entry point lazy-loads its diagram chunks over the network and is
therefore unusable here.

The bundle's last line publishes the global the app depends on:

    globalThis["mermaid"] = globalThis.__esbuild_esm_mermaid_nm["mermaid"].default;

Upgrading: replace mermaid.min.js, then update the version, date, size and sha256
above. Verify the global assignment above still exists — the app guards on
`window.mermaid` and degrades to showing diagram source without it.

Out of scope by nature: ELK layout (a separate package) and remote icon packs
(they fetch at render time).
